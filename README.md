# Автоматизация рабочего места водителя строительной техники

Программный комплекс для мониторинга наработки моточасов и контроля технического обслуживания (ТО) дорожно-строительной спецтехники (экскаваторы, бульдозеры, погрузчики).

## О проекте
Данное приложение разработано для упрощения ежедневной отчетности водителей и предотвращения поломок техники за счет своевременного сервисного обслуживания. Проект выполнен в рамках учебной дисциплины «Информатика» (ПНИПУ, 2025).

## Основные возможности
* **Учет моточасов:** Удобный ввод и хранение данных о наработке с автоматической фиксацией даты.
* **Контроль состояния узлов:** Отслеживание ресурса 7 ключевых компонентов (масла, фильтры, ремни, гидравлика).
* **Система оповещений:** Цветовая индикация состояния агрегатов:
    * **Синий** — ресурс в норме.
    * **Красный** — требуется немедленное обслуживание.
* **Архив работ:** История всех записей с возможностью добавления комментариев к выполненным операциям.
* **Безопасность:** Система авторизации и регистрации для защиты данных от несанкционированного доступа.

## Технологический стек
Проект реализован как кроссплатформенное десктопное приложение:
- **Язык программирования:** C++
- **Фреймворк:** Qt (использование Qt Widgets, QSqlDatabase)
- **База данных:** SQLite (локальное хранилище данных)
- **Архитектура:** Модульная структура с разделением логики авторизации, учета и визуализации.

## Интерфейс
Дизайн приложения оптимизирован для работы в кабине спецтехники:
1. **Эргономика:** Крупные кнопки и элементы управления для работы в условиях вибрации или на сенсорных экранах.
2. **Читаемость:** Минималистичный интерфейс, не отвлекающий от основных показателей.
3. **Интерактивность:** Плавные переходы между окнами и всплывающие уведомления.

## Структура данных (SQLite)
Приложение использует реляционную базу данных со следующими таблицами:
- `users` — учетные данные пользователей.
- `working_hours` — записи о наработке часов, датах и примечаниях.
